01Starting point

Already had a working pipeline — in Python

src/memory_graph/ already ran ingest → extract → store end to end from the terminal: ingest.py chunked notes, llm.py called Ollama, graph.py wrote MERGE upserts to Neo4j — backed by 7 passing tests. The goal for this phase wasn't to redesign the pipeline, it was to move it natively into Obsidian so ingestion didn't require a separate terminal step.

02De-risking

Proved the two hard connections first

Before writing any real plugin code, a throwaway spike answered two questions: can Obsidian's renderer reach Neo4j over bolt, and can it reach Ollama? Neo4j worked via the driver's browser/WebSocket build. Ollama worked via requestUrl(), which routes through Electron's main process — sidestepping renderer CORS entirely, no OLLAMA_ORIGINS environment variable required. Two commands, two green Notices, spike retired once its job was done.

03Build

Four checkpoints, tested in Obsidian between each

Every stage below was built, then confirmed working inside Obsidian — settings tab rendered, chunk counts sane, Ollama round-tripped, graph populated — before the next one started.

01

Scaffold + Settings

Plugin shell and a settings tab: Neo4j URI / user / password, Ollama URL / model, and a list of excluded folders.

02

Vault Reading

Via app.vault and app.metadataCache — never Node fs. Frontmatter stripped, chunked ~450 words by heading, strictly read-only.

03

Extraction

Ollama called via requestUrl() with format: json. Each returned triple is validated; malformed ones are skipped, never thrown.

04

Graph Writes

Uniqueness constraints, MERGE on Entity / REL / MENTIONED_IN, a content hash on every :Note for incremental skip, plus Ingest vault and Clear graph commands.

04Caught in testing

Three things the live run surfaced

Model quirk

Empty object instead of empty array

Ollama's format: json guarantees valid JSON, not the shape asked for — some chunks with no facts came back as {} rather than []. Now treated as "nothing here," not logged as an error.

Model quirk

A bare triple instead of a one-item array

Occasionally the model returned a single well-formed triple object at the top level instead of wrapping it in an array. Previously dropped as malformed — now accepted, since it isn't.

Pre-empted

Wikilinks would have fractured the graph

Flagged before it hit: an unstripped [[Anna]] would extract as a literal-bracketed entity, never matching the plain Anna found elsewhere in prose. Fixed with stripWikilinks(), applied to note body and title before chunking.

07What's next

Contradiction resolution, then the rest

→

Contradiction resolutionnext

The actual point of the project. Phase 1 stores conflicting facts as-is — e.g. a note placing someone in Lahore, a later one in Berlin — each keeping its own valid_from / last_seen. Deciding how those resolve is the open problem.

→

Wikilinks as their own signaldeferred

app.metadataCache.resolvedLinks already has every link in the vault parsed. A hand-typed link is a stronger signal than an LLM inference — a natural fit for confidence 1.0 edges that win by default. Cheap to add, intentionally scoped out until contradiction resolution lands.

→

Decay & Graph RAG queryingqueued

Temporal decay of unreasserted facts, and question-answering over the graph with citations — both out of scope until the above lands.
